## クイックスタート

Claude Code内で以下の3ステップを順番に実行してください。

### 1. マーケットプレイスの登録

BurnBoardプラグインリポジトリをマーケットプレイスとして登録します。

```bash
claude marketplace add https://github.com/behappybill/burnboard-plugin
```

### 2. プラグインのインストール

登録されたマーケットプレイスからBurnBoardプラグインをインストールします。

```bash
claude plugin install burnboard
```

インストール中にhook権限の承認を求められる場合があります。許可してください。

### 3. セットアップ

Claude Codeを起動した後、以下のスラッシュコマンドを入力します。

```
/burnboard:setup
```

APIキーの入力を求めるプロンプトが表示されます。[burnboard.io/settings](https://burnboard.io/settings)でGitHubログイン後にAPIキーを取得して入力してください。

セットアップが完了すると、トークン使用量が自動的に追跡されます。

## 主な機能

- **自動追跡** — セットアップ後は設定不要。すべてのClaude Codeセッションが自動的に記録されます
- **ステータスラインHUD** — ターミナルのステータスバーにリアルタイムのトークン使用量、ティア、進捗が表示されます
- **ゼロオーバーヘッド** — Stopフックは純粋なシェルスクリプト（~1ms）で、Claude Codeの応答速度に影響しません
- **オフライン対応** — レポート失敗時はローカルにキューイングされ、次のセッションで再試行されます
- **バッチレポート** — 複数の保留中のセッションを単一のAPI呼び出しで送信します

## ステータスラインHUD

セットアップ後、ターミナルのステータスバーにリアルタイムのトークン使用量が表示されます：

```
🔶 Ember  1.8M/5.0M ██░░░░░░░░ 19%  │  Session: 84.2K
```

- **ティアアイコン & 名前** — 現在のティアに応じた色分け表示
- **月間トークン / 次のティア閾値** — 進捗の追跡
- **プログレスバー** — 次のティアまでの視覚的な進捗表示
- **セッショントークン** — 現在のセッション使用量

ステータスラインはClaude CodeのStatusline APIを通じて自動更新され、パフォーマンスのために5秒キャッシュが適用されます。

## 動作の仕組み

```
Claude Codeセッション
       │
       ├─ [Stopイベント] ──→ mark.sh
       │                    セッションID + トランスクリプトパスを保存
       │                    純粋シェル、~1ms、Node.js起動なし
       │
       └─ [SessionEnd] ──→ flush.mjs
                            トランスクリプトJSONLをパース
                            モデル別トークン使用量を抽出
                            BurnBoard APIにレポート
                            成功時にクリーンアップ
```

プラグインは2つのClaude Codeフックを登録します：

| フック | スクリプト | 動作 |
|------|--------|-------------|
| `Stop` | `mark.sh` | セッションIDとトランスクリプトパスを記録（~1ms、純粋シェル） |
| `SessionEnd` | `flush.mjs` | トランスクリプトのパース、トークン抽出、APIレポート |

すべての保留中データはレポート成功まで`${CLAUDE_PLUGIN_DATA}`に保存されます。

## ティアシステム

毎月ランクを上げましょう：

**Spark** (0) → **Ember** (100万) → **Flame** (500万) → **Blaze** (2000万) → **Inferno** (5000万) → **Supernova** (1億+)

## プライバシー

- 集計されたトークン数のみ送信されます（入出力トークン、モデル、ターン数、所要時間）
- 会話の内容は送信されません
- 設定から公開リーダーボードへの参加を拒否できます

## ローカル開発

```bash
claude --plugin-dir ./
```
